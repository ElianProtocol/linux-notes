## 1. Что такое пакеты и зачем они нужны

**Пакет** — архив с файлами программы, её зависимостями и инструкциями по установке.  
**Репозиторий** — централизованное хранилище пакетов (как магазин приложений).

**APT** (Advanced Package Tool) — система управления пакетами в Debian:
- скачивает пакеты из репозиториев
- автоматически устанавливает **зависимости**
- обновляет и удаляет программы

> **Зависимости** — другие пакеты, необходимые для работы программы. APT устанавливает их автоматически.

## 2. Репозитории

### Файлы конфигурации

| Файл | Назначение |
|------|------------|
| `/etc/apt/sources.list` | Основной список репозиториев |
| `/etc/apt/sources.list.d/` | Дополнительные файлы с репозиториями |

### Структура строки репозитория

```
deb http://deb.debian.org/debian trixie main contrib non-free-firmware
│   │                        │      │
│   │                        │      └── кодовое имя версии ОС (trixie = Debian 13)
│   │                        └───────── секции репозитория
│   └─────────────────────────────────── URL репозитория
└─────────────────────────────────────── тип (deb = бинарные пакеты)
```

### Секции репозитория

| Секция | Описание |
|--------|----------|
| **main** | Свободное ПО, поддерживается Debian |
| **contrib** | Свободное ПО, зависящее от несвободных компонентов |
| **non-free** | Несвободное ПО (проприетарные драйверы) |
| **non-free-firmware** | Прошивки устройств (Wi-Fi, видеокарты) |

## 3. Основные команды apt

### Обновление списка пакетов

```bash
sudo apt update              # обновить «оглавление» репозиториев
```

> **Важно:** `update` не обновляет программы, только список доступных пакетов.

### Поиск и информация

| Команда | Описание |
|---------|----------|
| `apt search nginx` | Поиск по имени и описанию |
| `apt show nginx` | Подробная информация о пакете |
| `dpkg -l` | Список всех установленных пакетов |
| `dpkg -l \| grep nano` | Проверить, установлен ли пакет |

### Установка

```bash
sudo apt install tree        # установить пакет
sudo apt install -y tree     # без подтверждения
sudo apt install nginx php-fpm mariadb-server   # несколько пакетов
```

### Обновление

| Команда | Описание |
|---------|----------|
| `sudo apt upgrade` | Обновить все пакеты (без удаления) |
| `sudo apt full-upgrade` | Может удалять/добавлять пакеты (реже) |

### Удаление

| Команда | Описание |
|---------|----------|
| `sudo apt remove tree` | Удалить, конфиги остаются |
| `sudo apt purge tree` | Удалить полностью (с конфигами) |
| `sudo apt autoremove` | Удалить ненужные зависимости |

## 5. Практика: утилита tree

```bash
# Установка
sudo apt install -y tree

# Использование
tree ~/wordpress-project              # структура каталогов
tree -d ~/wordpress-project           # только каталоги
tree -p ~/wordpress-project           # с правами доступа
tree -L 1 ~/wordpress-project         # ограничить глубину
```

## 6. Важные моменты

### Зачем нужен sudo

Управление пакетами — системная операция. Требует прав суперпользователя (root).  
`sudo` позволяет выполнить команду с этими правами.

### apt update vs apt upgrade

| Команда | Что делает |
|---------|------------|
| `update` | Обновляет список доступных пакетов (каталог) |
| `upgrade` | Обновляет сами программы |

> **Правило:** Всегда сначала `update`, потом `upgrade`.

### remove vs purge

| Команда | Что делает | Когда использовать |
|---------|------------|-------------------|
| `remove` | Удаляет программу, оставляет конфиги | Удобно для переустановки |
| `purge` | Удаляет всё, включая конфигурацию | Полное удаление |

### Статусы в dpkg -l

| Статус | Значение |
|--------|----------|
| `ii` | Установлен и настроен |
| `rc` | Удалён, но конфиги остались |
| `pn` | Никогда не устанавливался |

## 7. Справочник команд

| Команда | Действие |
|---------|----------|
| `sudo apt update` | Обновить список пакетов |
| `sudo apt upgrade` | Обновить все пакеты |
| `sudo apt full-upgrade` | Обновить с разрешением удаления |
| `apt search <запрос>` | Поиск пакета |
| `apt show <пакет>` | Информация о пакете |
| `sudo apt install <пакет>` | Установить пакет |
| `sudo apt install -y <пакет>` | Установить без подтверждения |
| `sudo apt remove <пакет>` | Удалить пакет (конфиги остаются) |
| `sudo apt purge <пакет>` | Полностью удалить пакет |
| `sudo apt autoremove` | Удалить ненужные зависимости |
| `dpkg -l` | Список установленных пакетов |
| `dpkg -l \| grep <пакет>` | Проверить установку пакета |
| `sudo apt edit-sources` | Редактировать sources.list (проверяет синтаксис) |

## 8. Итоги

**Что нужно запомнить:**

| № | Что узнали | Ключевое |
|---|------------|----------|
| 1 | **Пакеты** | Способ распространения ПО в Linux |
| 2 | **APT** | Менеджер пакетов в Debian/Ubuntu |
| 3 | **Репозитории** | Хранилища пакетов. Список в `/etc/apt/sources.list` |
| 4 | **Секции** | `main` (свободное ПО), `contrib`, `non-free`, `non-free-firmware` |
| 5 | **Обязательная последовательность** | `sudo apt update` → `sudo apt upgrade` |
| 6 | **Основные команды** | `search`, `install`, `remove`, `autoremove` |
| 7 | **sudo** | Нужен для всех операций с пакетами (системные права) |

**📌 Ключевое правило администратора:** Перед установкой любого пакета всегда выполняйте `sudo apt update`, чтобы получить актуальный список пакетов из репозиториев.

**📌 Типичная ошибка:** Забыли `sudo` перед `apt install`

```bash
# ❌ БУДЕТ ОШИБКА
apt install tree
# E: Не удалось открыть файл блокировки... Permission denied

# ✅ ПРАВИЛЬНО
sudo apt install tree
```

**📌 Полезная команда для очистки диска:**

```bash
# Удалить ненужные пакеты и очистить кэш
sudo apt autoremove && sudo apt autoclean
```
