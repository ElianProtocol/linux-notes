## 1. Зачем в Linux несколько пользователей

| Причина | Описание |
|---------|----------|
| **Изоляция** | Если взломают один сервис, злоумышленник получит доступ только к его файлам |
| **Разграничение ответственности** | Каждый сервис работает от своего пользователя |
| **Аудит** | Все действия привязаны к конкретному пользователю |

> **Правило:** работайте под обычным пользователем, root-права запрашивайте только когда нужно.

---

## 2. Пользователь root

- Суперпользователь (UID 0)
- Может **всё**: читать любые файлы, завершать процессы, менять настройки
- Одна ошибка под root может уничтожить систему:

```bash
rm -rf /   # удалит ВСЮ систему (под обычным пользователем не выполнится)
```

---

## 3. Файлы пользователей и групп

### `/etc/passwd` — информация о пользователях

```bash
cat /etc/passwd
```

**Формат:**
```
имя:x:UID:GID:комментарий:домашний_каталог:оболочка
```

**Пример:**
```
student:x:1000:1000::/home/student:/bin/bash
www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin
```

### `/etc/shadow` — хеши паролей

Доступен только для root:

```bash
sudo cat /etc/shadow
```

### `/etc/group` — информация о группах

```bash
cat /etc/group
```

**Формат:**
```
имя_группы:x:GID:список_участников
```

**Пример:**
```
sudo:x:27:student
users:x:100:student,webmaster
```

### UID: системные vs обычные пользователи

| UID | Тип | Пример |
|-----|-----|--------|
| 0 | root | root |
| 1–999 | Системные | www-data, mysql, sshd |
| 1000+ | Обычные | student, webmaster |

---

## 4. Группы

- **Основная группа (primary)** — указана в `/etc/passwd`. Новые файлы принадлежат этой группе
- **Дополнительные группы (supplementary)** — дают дополнительные права

### Проверка принадлежности к группам

```bash
id                 # текущий пользователь
id webmaster       # конкретный пользователь
groups webmaster   # только группы
```

## 5. Управление пользователями

### Создание пользователя

```bash
sudo adduser webmaster          # интерактивно (рекомендуется)
```

> `adduser` — дружелюбная обёртка над `useradd`

### Смена пароля

```bash
sudo passwd webmaster   # сменить пароль другого пользователя
passwd                  # сменить свой пароль
```

### Изменение параметров пользователя

```bash
sudo usermod -aG sudo webmaster   # добавить в группу (ВАЖНО: -a обязателен!)
sudo usermod -L webmaster         # заблокировать (Lock)
sudo usermod -U webmaster         # разблокировать
```

> **Осторожно:** `usermod -G` без `-a` **заменит** все группы пользователя на указанные.

### Удаление пользователя

```bash
sudo deluser webmaster                # удалить пользователя
sudo deluser --remove-home webmaster  # удалить с домашним каталогом
```

---

## 6. Управление группами

| Команда | Описание |
|---------|----------|
| `sudo addgroup webdev` | Создать группу |
| `sudo delgroup webdev` | Удалить группу |
| `sudo adduser webmaster webdev` | Добавить пользователя в группу |

---

## 7. Переключение пользователей: su

| Команда | Описание |
|---------|----------|
| `su webmaster` | Переключиться (окружение остаётся старым) |
| `su - webmaster` | Переключиться с полной загрузкой окружения **(рекомендуется)** |
| `su -` | Переключиться на root (требуется пароль root) |

**Вернуться:** `Ctrl+D` или `exit`

---

## 8. sudo — выполнение команд от имени root

### Преимущества sudo

| Преимущество | Описание |
|--------------|----------|
| **Точечные права** | Root-права только на одну команду |
| **Аудит** | Все действия логируются |
| **Свой пароль** | Не нужен пароль root |

### Право на sudo

Пользователь получает право через членство в группе `sudo`:

```bash
sudo usermod -aG sudo webmaster
```

### Проверка sudo

```bash
sudo apt update   # выполнить команду от root
```

### Настройка sudo — `/etc/sudoers`

Редактировать **только** через `visudo` (проверяет синтаксис!):

```bash
sudo visudo
```

**Формат правил:**
```
%sudo   ALL=(ALL:ALL) ALL
│       │   │          │
│       │   │          └── любые команды
│       │   └── от имени любого пользователя и группы
│       └── на любом хосте
└── группа sudo (знак % означает группу)
```

### Каталог `/etc/sudoers.d/`

Вместо правки основного файла можно создавать отдельные файлы в этом каталоге.

## 9. Практика: настройка пользователя для проекта

### Шаг 1. Создаём группу и пользователя

```bash
sudo addgroup webdev
sudo adduser webmaster          # задать пароль
```

### Шаг 2. Добавляем в группы

```bash
sudo adduser webmaster webdev   # группа проекта
sudo adduser webmaster sudo     # право на sudo
```

### Шаг 3. Меняем владельца проекта

```bash
sudo chown -R webmaster:webdev ~/wordpress-project
```

### Шаг 4. Открываем проход в домашний каталог

```bash
chmod o+x ~                     # только x (проход), без r (просмотр)
```

### Шаг 5. Настраиваем права для группы

```bash
sudo chmod -R g+rwX ~/wordpress-project   # заглавная X = только для каталогов
```

### Шаг 6. Проверка

```bash
su - webmaster
cd ~student/wordpress-project
ls -la
cat config/nginx.conf
echo "test" >> site/index.html
exit
```

## 10. Справочник команд

### Управление пользователями (команды)

| Команда | Описание |
|---------|----------|
| `sudo adduser имя` | Создать пользователя (интерактивно) |
| `sudo deluser имя` | Удалить пользователя |
| `sudo deluser --remove-home имя` | Удалить с домашним каталогом |
| `sudo passwd имя` | Сменить пароль |
| `sudo usermod -aG группа имя` | Добавить в группу |
| `sudo usermod -L имя` | Заблокировать |
| `sudo usermod -U имя` | Разблокировать |

### Управление группами (команды)

| Команда | Описание |
|---------|----------|
| `sudo addgroup имя` | Создать группу |
| `sudo delgroup имя` | Удалить группу |
| `sudo adduser пользователь группа` | Добавить пользователя в группу |

### Просмотр информации (команды)

| Команда | Описание |
|---------|----------|
| `id имя` | Информация о пользователе |
| `groups имя` | Группы пользователя |
| `cat /etc/passwd` | Список пользователей |
| `cat /etc/group` | Список групп |

### Переключение и sudo (команды)

| Команда | Описание |
|---------|----------|
| `su - имя` | Переключиться на пользователя |
| `sudo команда` | Выполнить команду от root |
| `sudo visudo` | Редактировать /etc/sudoers (безопасно) |

## 11. Итоги

**Что нужно запомнить:**

| № | Что узнали | Ключевое |
|---|------------|----------|
| 1 | **Основное правило** | Работайте под обычным пользователем, root — только через `sudo` |
| 2 | **Пользователи** | `adduser`, `deluser`, `passwd`, `usermod` |
| 3 | **Группы** | `addgroup`, `delgroup`, `adduser пользователь группа` |
| 4 | **Проверка** | `id`, `groups`, `cat /etc/passwd`, `cat /etc/group` |
| 5 | **sudo** | Членство в группе `sudo`, настройка через `visudo` |
| 6 | **Переключение** | `su - имя` (с дефисом!) |
| 7 | **Права доступа** | Для доступа к вложенному каталогу достаточно `x` на родительском (`chmod o+x ~`) |
| 8 | **Заглавная X** | В `chmod` добавляет `x` только каталогам и файлам, где уже есть бит выполнения |

**📌 Ключевое правило администратора:** Никогда не работайте под root без необходимости. Для разовых команд используйте `sudo`, для длительной работы — `sudo -i` или `su -`.

**📌 Ключевая команда для проверки прав:**

```bash
# Проверить, какие команды может выполнять пользователь через sudo
sudo -l
```

**📌 Типичная ошибка:** Забыли `-a` в `usermod -aG`

```bash
# ❌ ПЛОХО — заменит все группы на sudo
sudo usermod -G sudo webmaster

# ✅ ХОРОШО — добавит sudo к существующим группам
sudo usermod -aG sudo webmaster
```
