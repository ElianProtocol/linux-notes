## 1. Зачем разбираться в сети

Для работы веб-сервера необходимо понимать:

- какой **IP-адрес** у вашего сервера;
- какие **порты** открыты и какие программы их слушают;
- как **защитить** сервер файрволом;
- как работает **DNS** (преобразование доменных имён в IP-адреса).

Без этих знаний невозможно ни настроить веб-сервер, ни диагностировать проблемы с доступностью сайта.

## 2. Основы IP-сетей

### IP-адреса

IP-адрес — это четыре числа от 0 до 255, разделённые точками: `192.168.1.100`

**Частные (локальные) диапазоны** — не маршрутизируются в интернете:

| Диапазон | Количество адресов |
|----------|---------------------|
| `10.0.0.0 – 10.255.255.255` | 16,7 млн |
| `172.16.0.0 – 172.31.255.255` | 1,05 млн |
| `192.168.0.0 – 192.168.255.255` | 65 536 |

**Специальные адреса:**

| Адрес | Назначение |
|-------|------------|
| `127.0.0.1` (localhost) | «Сам компьютер», всегда указывает на текущую машину |
| `0.0.0.0` | «Все интерфейсы» (сервер слушает на всех IP) |
| `255.255.255.255` | Широковещательный адрес |

### Маска подсети и CIDR

Маска определяет, какая часть IP-адреса — сеть, а какая — устройство.

| Маска | CIDR | Количество устройств |
|-------|------|----------------------|
| `255.255.255.0` | `/24` | 254 |
| `255.255.0.0` | `/16` | 65 534 |
| `255.0.0.0` | `/8` | 16,7 млн |

**Пример:** `192.168.1.100/24` — адрес в сети `192.168.1.x`

### Шлюз (gateway)

«Дверь наружу» из локальной сети. Обычно это маршрутизатор — `192.168.1.1`. Все пакеты, адресованные за пределы локальной сети, отправляются через шлюз.

### Порты

Порт — это число от 1 до 65535, которое идентифицирует конкретную программу или сервис на компьютере.

| Диапазон | Назначение |
|----------|------------|
| 1–1023 | Известные (системные) порты |
| 1024–49151 | Зарегистрированные порты |
| 49152–65535 | Динамические (временные) |

**Важные порты для администратора:**

| Порт | Сервис | Протокол |
|------|--------|----------|
| 22 | SSH | TCP |
| 80 | HTTP | TCP |
| 443 | HTTPS | TCP |
| 3306 | MySQL/MariaDB | TCP |
| 25 | SMTP (почта) | TCP |
| 53 | DNS | UDP/TCP |

> **Аналогия:** IP-адрес = адрес многоквартирного дома, порт = номер квартиры.

### TCP и UDP

| Характеристика | TCP | UDP |
|----------------|-----|-----|
| Соединение | Устанавливает (handshake) | Без соединения |
| Надёжность | Гарантирует доставку | Не гарантирует |
| Порядок пакетов | Сохраняет | Не сохраняет |
| Контроль ошибок | Да | Нет |
| Скорость | Ниже | Выше |
| **Примеры** | SSH, HTTP, базы данных | DNS, видео, игры |

## 3. ip — сетевые интерфейсы и адреса

### Просмотр IP-адресов

```bash
ip addr show      # полная версия
ip a              # сокращённая форма
```

**Пример вывода:**
```
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN
    inet 127.0.0.1/8 scope host lo
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast state UP
    inet 192.168.1.100/24 brd 192.168.1.255 scope global eth0
```

| Элемент | Значение |
|---------|----------|
| `lo` | Loopback-интерфейс (127.0.0.1) |
| `eth0`, `ens3`, `enp0s3` | Физический сетевой интерфейс |
| `UP` | Интерфейс включён и работает |
| `LOWER_UP` | Кабельное соединение есть |
| `inet 192.168.1.100/24` | IP-адрес и маска подсети |
| `brd` | Широковещательный адрес |

### Просмотр маршрутов

```bash
ip route show    # полная
ip r             # сокращённая
```

**Пример вывода:**
```
default via 192.168.1.1 dev eth0          # шлюз по умолчанию
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.100
```

| Элемент | Значение |
|---------|----------|
| `default via` | IP-адрес шлюза по умолчанию |
| `dev eth0` | Через какой интерфейс идти |
| `proto kernel` | Маршрут добавлен ядром автоматически |
| `scope link` | Адрес доступен напрямую (в той же сети) |

### Просмотр состояния интерфейсов

```bash
ip link show                # все интерфейсы
ip link set eth0 up/down    # включить/выключить интерфейс
```

## 4. ss — сетевые соединения

### Какие порты слушают программы

```bash
ss -tlnp         # TCP, слушающие, порты, процессы
sudo ss -tlnp    # с процессами всех пользователей
```

**Флаги:**

| Флаг | Значение |
|------|----------|
| `-t` | только TCP |
| `-u` | только UDP |
| `-l` | только слушающие (listening) |
| `-n` | номера портов (не преобразовывать в имена) |
| `-p` | показать процесс (требует sudo) |
| `-a` | все сокеты (и слушающие, и установленные) |

**Пример вывода:**
```
State   Recv-Q  Send-Q  Local Address:Port  Peer Address:Port  Process
LISTEN  0       128     0.0.0.0:22          0.0.0.0:*          users:(("sshd",pid=654,fd=3))
LISTEN  0       128     0.0.0.0:80          0.0.0.0:*          users:(("nginx",pid=1234,fd=6))
```

| Элемент | Значение |
|---------|----------|
| `LISTEN` | Сокет ожидает входящие соединения |
| `Recv-Q` | Данные в очереди на приём |
| `Send-Q` | Данные в очереди на отправку |
| `0.0.0.0:80` | Слушает на всех интерфейсах, порт 80 |
| `127.0.0.1:3306` | Слушает только на localhost |
| `users:(("nginx",...` | Программа, которая слушает порт |

### Установленные соединения

```bash
ss -tn          # активные TCP-соединения (без -l)
ss -tun         # TCP + UDP, без преобразования портов
```

**Пример вывода:**
```
State   Recv-Q  Send-Q  Local Address:Port   Peer Address:Port
ESTAB   0       0       192.168.1.100:22     192.168.1.50:54321
```

| Состояние | Значение |
|-----------|----------|
| `ESTAB` | Соединение установлено |
| `TIME_WAIT` | Соединение закрывается |
| `CLOSE_WAIT` | Ожидание закрытия от приложения |

## 5. Проверка соединений

### ping — проверка доступности хоста

```bash
ping 8.8.8.8               # бесконечно (Ctrl+C для остановки)
ping -c 3 8.8.8.8          # отправить 3 пакета и завершить
ping google.com            # с разрешением имени через DNS
```

**Что даёт ping:**
- Проверяет, отвечает ли удалённый хост
- Измеряет время ответа (RTT — Round Trip Time)
- Показывает потери пакетов

**Пример вывода:**
```
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=15.2 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=117 time=14.8 ms

--- 8.8.8.8 ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1001ms
rtt min/avg/max/mdev = 14.8/15.0/15.2/0.2 ms
```

### curl — HTTP-запросы

```bash
curl http://example.com              # показать HTML-код страницы
curl -I http://example.com           # только заголовки ответа
curl -s http://example.com           # тихий режим (без прогресса)
curl -o /dev/null -w "%{http_code}"  # получить только код ответа
```

**Пример заголовков:**
```
HTTP/1.1 200 OK
Content-Type: text/html; charset=UTF-8
Content-Length: 1256
Server: nginx/1.18.0
```

### Другие инструменты

```bash
# Трассировка маршрута (показывает путь пакетов)
traceroute google.com

# Проверка открытого порта
nc -zv google.com 80
```

## 6. DNS — разрешение имён

### Как работает DNS

DNS (Domain Name System) преобразует доменные имена (`google.com`) в IP-адреса (`142.250.74.46`).

**Процесс разрешения имени:**
1. Приложение запрашивает `google.com`
2. Система проверяет `/etc/hosts`
3. Если нет — идёт запрос к DNS-серверу из `/etc/resolv.conf`
4. DNS-сервер возвращает IP-адрес

### Файлы конфигурации DNS

| Файл | Назначение |
|------|------------|
| `/etc/resolv.conf` | DNS-сервер, к которому обращается система |
| `/etc/hosts` | Локальные статические записи (проверяется **раньше** DNS) |

```bash
cat /etc/resolv.conf
# nameserver 192.168.1.1
# nameserver 8.8.8.8

cat /etc/hosts
# 127.0.0.1       localhost
# 192.168.1.10    myserver.local
```

### Проверка DNS

```bash
# Простой запрос
host google.com
# google.com has address 142.250.74.46

# Подробный запрос (требуется установка: apt install dnsutils)
dig google.com

# Обратный запрос (IP → имя)
host 8.8.8.8
# 8.8.8.8.in-addr.arpa domain name pointer dns.google.

# Указать конкретный DNS-сервер
host google.com 8.8.8.8
```

## 7. ufw — файрвол

### Зачем нужен

Файрвол контролирует, какой сетевой трафик разрешён, а какой запрещён. По умолчанию в Linux всё открыто — ufw позволяет быстро настроить ограничения.

> **ufw** (Uncomplicated Firewall) — удобная обёртка над iptables/nftables.

### Установка и включение

```bash
sudo apt install ufw -y

# ⚠️ ВАЖНО: сначала разрешить SSH!
sudo ufw allow 22
sudo ufw allow 22/tcp

# Теперь можно включать
sudo ufw enable
```

> **Внимание!** Если не разрешить порт 22 ДО включения файрвола, вы потеряете SSH-доступ к серверу. При работе на удалённом сервере это критично.

### Управление правилами

```bash
# Статус и правила
sudo ufw status                 # статус и список правил
sudo ufw status verbose         # подробный статус
sudo ufw status numbered        # с номерами (для удаления)

# Разрешение портов
sudo ufw allow 80               # разрешить порт 80
sudo ufw allow 80/tcp           # явно указать TCP
sudo ufw allow http             # по имени сервиса (из /etc/services)
sudo ufw allow from 192.168.1.0/24   # разрешить целую сеть

# Запрет портов
sudo ufw deny 3306              # запретить порт 3306
sudo ufw deny 22                # ⚠️ заблокирует SSH — делайте осознанно

# Удаление правил
sudo ufw delete allow 80        # удалить правило по описанию
sudo ufw delete 3               # удалить правило номер 3

# Управление файрволом
sudo ufw disable                # отключить (правила сохраняются)
sudo ufw reset                  # сбросить все правила
sudo ufw reload                 # перезагрузить правила
```

### Статусы и логи

```bash
# Более детальный статус
sudo ufw status verbose

# Логи файрвола
sudo tail -f /var/log/ufw.log
```

## 8. Практика: настройка сети для проекта

### Шаг 1. Узнаём IP-адрес сервера

```bash
ip addr show
# или
hostname -I
```

Запишите адрес сетевого интерфейса (не `127.0.0.1`) — он понадобится для проверки доступности сайта.

### Шаг 2. Проверяем шлюз и доступ в интернет

```bash
# Смотрим шлюз
ip route show | grep default

# Проверяем связь с внешним миром
ping -c 3 8.8.8.8
ping -c 3 google.com  # проверяет и DNS
```

### Шаг 3. Смотрим, какие порты слушают наши сервисы

```bash
sudo ss -tlnp | grep -E "(nginx|php|mysql|ssh)"
```

Должны увидеть:
- `sshd` на порту 22
- `nginx` на порту 80 (и 443, если настроен HTTPS)
- `php-fpm` на сокете или порту 9000
- `mariadb` на порту 3306 (обычно слушает только 127.0.0.1)

### Шаг 4. Настраиваем файрвол для веб-сервера

```bash
# Установка
sudo apt install ufw -y

# Разрешаем нужные порты
sudo ufw allow 22/tcp      # SSH
sudo ufw allow 80/tcp      # HTTP
sudo ufw allow 443/tcp     # HTTPS (если настроите)

# Проверяем правила перед включением
sudo ufw show added

# Включаем
sudo ufw enable

# Проверяем статус
sudo ufw status verbose
```

### Шаг 5. Проверяем DNS

```bash
# Какой DNS-сервер использует система
cat /etc/resolv.conf

# Проверяем разрешение имён
host google.com
host $(hostname -I | awk '{print $1}')  # обратный запрос для вашего IP
```

| Команда | Описание |
|---------|----------|

| Команда | Описание |
|---------|----------|

| `cat /etc/resolv.conf` | Показать DNS-сервер системы |
| `cat /etc/hosts` | Показать локальные DNS-записи |

### ufw — файрвол

| Команда | Описание |
|---------|----------|
| `sudo ufw status` | Статус и список правил |
| `sudo ufw status numbered` | Статус с номерами правил |
| `sudo ufw allow порт` | Разрешить порт |
| `sudo ufw allow from IP` | Разрешить конкретный IP |
| `sudo ufw deny порт` | Запретить порт |
| `sudo ufw delete номер` | Удалить правило по номеру |
| `sudo ufw enable` | Включить файрвол |
| `sudo ufw disable` | Отключить файрвол |
| `sudo ufw reload` | Перезагрузить правила |
| `sudo ufw reset` | Сбросить все правила |

## 10. Итоги

1. **IP-адрес и шлюз:**
   - `ip a` — посмотреть IP-адреса
   - `ip r` — посмотреть шлюз (строка `default via`)

2. **Порты для веб-сервера:**
   - `22` — SSH (обязательно открыть до включения ufw)
   - `80` — HTTP
   - `443` — HTTPS

3. **Проверка работающих сервисов:**
   - `sudo ss -tlnp` — посмотреть, какие порты слушают процессы

4. **Диагностика сети:**
   - `ping` — проверка доступности
   - `curl -I` — проверка HTTP-ответа

5. **DNS:**
   - `/etc/resolv.conf` — DNS-сервер системы
   - `/etc/hosts` — локальные записи (проверяются в первую очередь)
   - `host` — проверка разрешения имён

6. **Файрвол ufw — последовательность действий:**
   ```bash
   sudo ufw allow 22        # 1. Сначала разрешить SSH
   sudo ufw allow 80        # 2. Разрешить HTTP
   sudo ufw allow 443       # 3. Разрешить HTTPS (опционально)
   sudo ufw enable          # 4. Включить файрвол
   sudo ufw status          # 5. Проверить
   ```

7. **Типичная конфигурация файрвола для веб-сервера:**
   ```bash
   sudo ufw default deny incoming    # запретить всё входящее
   sudo ufw default allow outgoing   # разрешить всё исходящее
   sudo ufw allow 22/tcp
   sudo ufw allow 80/tcp
   sudo ufw allow 443/tcp
   sudo ufw enable
   ```
