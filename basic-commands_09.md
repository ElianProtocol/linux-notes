# Конспект: Сеть — ip, ss, ufw, основы DNS

## 1. Зачем разбираться в сети

Для работы веб-сервера необходимо понимать:
- IP-адрес сервера
- Какие порты открыты и кто их слушает
- Как защитить сервер файрволом
- Как работает DNS (преобразование имён в адреса)

---

## 2. Основы IP-сетей (врезка)

### IP-адреса
Четыре числа от 0 до 255, разделённые точками: `192.168.1.100`

| Диапазон | Тип |
|----------|-----|
| `10.0.0.0 – 10.255.255.255` | Частный |
| `172.16.0.0 – 172.31.255.255` | Частный |
| `192.168.0.0 – 192.168.255.255` | Частный |
| Всё остальное | Публичный |

- **127.0.0.1 (localhost)** — «сам компьютер», всегда указывает на текущую машину

### Маска подсети и CIDR
Определяет, какая часть адреса — сеть, какая — устройство.

```
255.255.255.0  =  /24   (до 254 устройств)
192.168.1.0/24 — сеть 192.168.1.x
```

### Шлюз (gateway)
«Дверь наружу» из локальной сети. Обычно маршрутизатор (`192.168.1.1`).

### Порты
Число от 1 до 65535. Идентифицирует программу на компьютере.

| Порт | Сервис |
|------|--------|
| 22 | SSH |
| 80 | HTTP |
| 443 | HTTPS |
| 3306 | MySQL/MariaDB |

> Аналогия: IP = адрес здания, порт = номер квартиры.

### TCP и UDP

| Протокол | Особенности | Примеры |
|----------|-------------|---------|
| **TCP** | Надёжный, гарантирует доставку | SSH, HTTP, базы данных |
| **UDP** | Быстрый, без гарантий | DNS, видео, игры |

## 3. ip — сетевые интерфейсы и адреса

### Просмотр IP-адресов
```bash
ip addr show      # полная версия
ip a              # сокращённая
```

Вывод:
```
1: lo: ... inet 127.0.0.1/8 scope host lo
2: eth0: ... inet 192.168.1.100/24 brd 192.168.1.255 scope global eth0
```

| Элемент | Значение |
|---------|----------|
| `lo` | Loopback (127.0.0.1) |
| `eth0` | Сетевой интерфейс (может быть ens3, enp0s3...) |
| `UP` | Интерфейс включён |
| `inet 192.168.1.100/24` | IP-адрес и маска |

### Просмотр маршрутов
```bash
ip route show    # полная
ip r             # сокращённая
```

```
default via 192.168.1.1 dev eth0          # шлюз по умолчанию
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.100
```

### Просмотр интерфейсов
```bash
ip link show     # состояние интерфейсов (UP/DOWN) без IP
```

## 4. ss — сетевые соединения

### Какие порты слушают
```bash
ss -tlnp         # TCP, слушающие, номера портов, процессы
sudo ss -tlnp    # с процессами всех пользователей
```

| Флаг | Значение |
|------|----------|
| `-t` | только TCP |
| `-l` | только слушающие (listening) |
| `-n` | номера портов (не имена) |
| `-p` | показать процесс (требует sudo) |

Пример:
```
State   Recv-Q  Send-Q  Local Address:Port  Process
LISTEN  0       128     0.0.0.0:22           0.0.0.0:*   users:(("sshd",pid=654,fd=3))
```

### Установленные соединения
```bash
ss -tn          # активные TCP-соединения (без -l)
```

## 5. Проверка соединений

### ping — доступность хоста
```bash
ping 8.8.8.8               # бесконечно (Ctrl+C остановить)
ping -c 3 8.8.8.8          # 3 пакета и завершить
ping google.com            # с разрешением имени
```

### curl — HTTP-запросы
```bash
curl http://example.com        # показать HTML-код
curl -I http://example.com     # только заголовки ответа
```

```
HTTP/1.1 200 OK
Content-Type: text/html; charset=UTF-8
```

## 6. DNS — разрешение имён

### Как работает
DNS преобразует доменные имена (`google.com`) в IP-адреса (`142.250.74.46`).

### Файлы конфигурации

| Файл | Назначение |
|------|------------|
| `/etc/resolv.conf` | DNS-сервер, к которому обращается система |
| `/etc/hosts` | Локальные записи (проверяется **раньше** DNS) |

```bash
cat /etc/resolv.conf     # nameserver 192.168.1.1
cat /etc/hosts           # 127.0.0.1 localhost
```

### Проверка DNS
```bash
host google.com          # простой запрос
dig google.com           # подробный (если установлен)
```

## 7. ufw — файрвол

### Зачем нужен
Контролирует, какой сетевой трафик разрешён. По умолчанию всё открыто — ufw позволяет ограничить доступ.

> **ufw** (Uncomplicated Firewall) — удобная обёртка над nftables.

### Установка и включение
```bash
sudo apt install ufw
sudo ufw allow 22        # ОБЯЗАТЕЛЬНО разрешить SSH перед включением!
sudo ufw enable
```

> **Внимание!** Если не разрешить порт 22 перед включением — потеряете доступ к серверу.

### Управление правилами
```bash
sudo ufw status                 # статус и правила
sudo ufw status numbered        # с номерами

sudo ufw allow 80               # разрешить порт 80
sudo ufw allow 80/tcp           # явно указать TCP
sudo ufw allow http             # по имени сервиса

sudo ufw deny 3306              # запретить порт

sudo ufw delete allow 80        # удалить правило по описанию
sudo ufw delete 3               # удалить правило номер 3

sudo ufw disable                # отключить (правила сохраняются)
```

## 8. Практика: настройка сети для проекта

### Шаг 1. Узнаём IP-адрес
```bash
ip addr show
```
Запишите адрес сетевого интерфейса (не 127.0.0.1) — понадобится в главе 11.

### Шаг 2. Проверяем шлюз и интернет
```bash
ip route show
ping -c 3 8.8.8.8
```

### Шаг 3. Смотрим открытые порты
```bash
sudo ss -tlnp
```
Должны увидеть `sshd` на порту 22.

### Шаг 4. Настраиваем файрвол
```bash
sudo apt install ufw
sudo ufw allow 22
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable
sudo ufw status
```

### Шаг 5. Проверяем DNS
```bash
host google.com
cat /etc/resolv.conf
```

## 9. Шпаргалка

### ip
| Команда | Описание |
|---------|----------|
| `ip a` | IP-адреса интерфейсов |
| `ip r` | Таблица маршрутов (шлюз) |
| `ip link show` | Состояние интерфейсов |

### ss
| Команда | Описание |
|---------|----------|
| `sudo ss -tlnp` | Слушающие TCP-порты с процессами |
| `ss -tn` | Активные TCP-соединения |

### Диагностика
| Команда | Описание |
|---------|----------|
| `ping -c 3 адрес` | Проверить доступность |
| `curl -I URL` | HTTP-заголовки ответа |

### DNS
| Команда | Описание |
|---------|----------|
| `host имя` | Узнать IP по имени |
| `cat /etc/resolv.conf` | DNS-сервер системы |
| `cat /etc/hosts` | Локальные DNS-записи |

### ufw
| Команда | Описание |
|---------|----------|
| `sudo ufw status` | Статус и правила |
| `sudo ufw allow порт` | Разрешить порт |
| `sudo ufw deny порт` | Запретить порт |
| `sudo ufw enable/disable` | Включить/отключить |
| `sudo ufw delete номер` | Удалить правило |

## 10. Итог: что нужно запомнить

1. **IP-адрес** — `ip a`; **шлюз** — `ip r` (default via)
2. **Порты:** 22 (SSH), 80 (HTTP), 443 (HTTPS)
3. **Слушающие порты:** `sudo ss -tlnp` (проверка, что сервис работает)
4. **Проверка связи:** `ping`, `curl -I`
5. **DNS:** `/etc/resolv.conf` (сервер), `/etc/hosts` (локальные записи)
6. **Файрвол ufw:** перед включением **обязательно разрешить SSH**:
   ```bash
   sudo ufw allow 22 && sudo ufw enable
   ```
7. **Типичная конфигурация для веб-сервера:**
   ```bash
   sudo ufw allow 22
   sudo ufw allow 80
   sudo ufw allow 443
   sudo ufw enable
   ```
