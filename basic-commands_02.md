## 1. Зачем это нужно

Linux — многопользовательская система. Права доступа определяют, **кто** и **что** может делать с файлами/каталогами, защищая их от случайного или намеренного вмешательства других пользователей.

---

## 2. Три категории пользователей

| Категория | Буква | Описание |
|-----------|-------|----------|
| **Владелец** | `u` (user) | Создатель файла |
| **Группа** | `g` (group) | Набор пользователей с общим доступом |
| **Остальные** | `o` (others) | Все остальные пользователи |

---

## 3. Три типа прав (`rwx`)

| Право | Буква | Значение для файла | Значение для каталога |
|-------|-------|-----------|-------------|
| **Чтение** | `r` (read) | Просмотр содержимого | Просмотр списка файлов (`ls`) |
| **Запись** | `w` (write) | Изменение содержимого | Создание/удаление файлов внутри |
| **Выполнение** | `x` (execute) | Запуск как программы | Вход в каталог (`cd`) |

> **Важно:** Права для каталогов работают иначе. Чтобы попасть в каталог, нужно право `x`. Чтобы увидеть список файлов — `r`. `w` без `x` бесполезен.

---

## 4. Как читать строку прав (`ls -l`)

```
-rw-r--r--
│  │    │
│  │    └── Остальные: r-- (только чтение)
│  └─────── Группа:     r-- (только чтение)
└────────── Владелец:   rw- (чтение и запись)
```

| Позиция | Что означает |
|---------|--------------|
| 1-й символ | Тип файла (`-` — файл, `d` — каталог) |
| 2-4 символы | Права для владельца |
| 5-7 символы | Права для группы |
| 8-10 символы | Права для остальных |

---

## 5. Числовой (восьмеричный) формат

Каждому праву соответствует число: `r=4`, `w=2`, `x=1`. Значения складываются.

| Комбинация | Сумма |
|------------|-------|
| `rwx` | 7 |
| `rw-` | 6 |
| `r-x` | 5 |
| `r--` | 4 |
| `---` | 0 |

**Стандартные комбинации (запомнить):**

| Права | Число | Для чего |
|-------|-------|----------|
| `rw-r--r--` | **644** | **Файлы** (владелец читает/пишет, остальные только читают) |
| `rwxr-xr-x` | **755** | **Каталоги и скрипты** (владелец всё может, остальные читают и входят) |

## 6. Команды для управления правами

### chmod — изменение прав

**Символьный формат:**

```bash
chmod u+x file        # добавить выполнение владельцу
chmod go-w file       # убрать запись у группы и остальных
chmod a+r file        # добавить чтение для всех (a = all)
chmod u=rwx,go=r file # установить точные права
```

**Числовой формат:**

```bash
chmod 644 file        # стандарт для файла
chmod 755 directory/  # стандарт для каталога
chmod -R 755 dir/     # рекурсивно (осторожно!)
```

### chown — смена владельца и группы

```bash
chown user file           # сменить владельца
chown user:group file     # сменить владельца и группу
chown :group file         # сменить только группу
chown -R user:group dir/  # рекурсивно
```

### chgrp — смена группы

```bash
chgrp group file          # сменить только группу
chgrp -R group dir/       # рекурсивно
```

---

## 7. Важные нюансы

| Нюанс | Пояснение |
|-------|-----------|
| **`chmod 777` — антипаттерн** | Снимает все ограничения, создавая дыры в безопасности. Вместо этого определите нужного пользователя (`www-data`), назначьте владельца (`chown`) и выставьте минимальные права (`644`/`755`) |
| **Менять владельца может только root** | `chown` доступен только суперпользователю |
| **Обычный пользователь меняет права только на свои файлы** | `chmod` работает только на файлах, где пользователь является владельцем |

---

## 8. Разные права на файлы и каталоги (паттерн)

Чтобы рекурсивно выставить разные права на файлы и папки, используют `find`:

```bash
# Права на каталоги: 755 (rwxr-xr-x)
find /path -type d -exec chmod 755 {} +

# Права на файлы: 644 (rw-r--r--)
find /path -type f -exec chmod 644 {} +
```

---

## 9. Специальные биты (обзор)

| Бит | Буква | Где появляется | Что делает |
|-----|-------|----------------|-------------|
| **SUID** | `s` | Вместо `x` у владельца | Выполнение от имени владельца файла |
| **SGID** | `s` | Вместо `x` у группы | Выполнение от имени группы; на каталоге — наследование группы |
| **Sticky bit** | `t` | Вместо `x` у остальных | Удалять файлы может только владелец (как в `/tmp`) |

```bash
# Установка специальных битов
chmod u+s file        # SUID
chmod g+s directory/  # SGID
chmod +t directory/   # Sticky bit
```

## 10. Справочник команд

| Команда | Описание |
|---------|----------|
| `ls -l` | Показать права доступа |
| `chmod u+x file` | Добавить выполнение владельцу |
| `chmod 644 file` | Установить права `rw-r--r--` |
| `chmod 755 dir` | Установить права `rwxr-xr-x` |
| `chmod -R 755 dir` | Рекурсивно установить права |
| `chown user file` | Сменить владельца |
| `chown user:group file` | Сменить владельца и группу |
| `chgrp group file` | Сменить группу |
| `chown -R user:group dir` | Рекурсивно сменить владельца и группу |

## 11. Итоги

**Что нужно запомнить:**

| № | Что узнали | Ключевое |
|---|------------|----------|
| 1 | **Категории** | Владелец (u), Группа (g), Остальные (o) |
| 2 | **Типы прав** | Чтение (r=4), Запись (w=2), Выполнение (x=1) |
| 3 | **Стандарт для файлов** | `chmod 644 file` |
| 4 | **Стандарт для каталогов** | `chmod 755 dir` |
| 5 | **Смена владельца** | `chown user:group file` (только root) |
| 6 | **Смена прав** | `chmod` (владелец может менять на своих файлах) |
| 7 | **Рекурсивная смена** | `chmod -R 755 dir/` (осторожно!) |

**📌 Ключевое правило администратора:** Никогда не используйте `chmod 777`. Вместо этого определите правильного владельца (`chown`) и выставьте минимальные необходимые права (`644` для файлов, `755` для каталогов).

**📌 Типичная ошибка:** Забыли про право `x` для каталога

```bash
# ❌ НЕПРАВИЛЬНО — в каталог нельзя зайти (cd не работает)
chmod 644 directory/

# ✅ ПРАВИЛЬНО — для каталога нужно x
chmod 755 directory/
```

**📌 Полезная команда для проверки прав:**

```bash
# Показать права в числовом формате
stat -c "%a %n" file.txt
# Вывод: 644 file.txt
```
