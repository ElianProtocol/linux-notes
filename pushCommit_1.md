# Конспект: Работа с коммитами и пушами в Git

## 1. Что такое коммит?

**Коммит (commit)** — это "снимок" состояния ваших файлов в определенный момент времени. Каждый коммит имеет:
- Уникальный хеш (идентификатор)
- Сообщение, описывающее изменения
- Автора и дату

```
Коммит = Сохранение + Описание изменений
```

---

## 2. Основные команды для коммитов

### Проверить статус
```bash
git status
```
Показывает, какие файлы изменены, добавлены или готовы к коммиту.

### Добавить файлы в коммит
```bash
# Добавить конкретный файл
git add filename.md

# Добавить все файлы в текущей папке
git add .

# Добавить все файлы во всем проекте
git add -A
```

### Сделать коммит
```bash
git commit -m "Краткое описание изменений"
```

### Примеры сообщений для конспектов
```bash
git commit -m "Добавлен конспект: основные команды Linux"
git commit -m "Обновлен конспект: добавлена информация о grep"
git commit -m "Исправлена опечатка в команде chmod"
git commit -m "Добавлены практические задания по навигации"
```

---

## 3. Что такое пуш (push)?

**Push** — это отправка ваших локальных коммитов на удаленный репозиторий (GitHub).

```
Локальный репозиторий → Push → GitHub (удаленный репозиторий)
```

### Основная команда
```bash
git push
```

### Первый push (связывание веток)
```bash
git push -u origin main
# или
git push -u origin master
```

- `-u` — устанавливает связь между локальной и удаленной веткой
- `origin` — имя удаленного репозитория
- `main/master` — имя вашей ветки

## 4. Полный рабочий процесс

```bash
# 1. Перейти в папку с репозиторием
cd ~/Linux-notes

# 2. Создать или изменить файлы

git add .

# 5. Сделать коммит
git commit -m "Добавлен конспект: основные команды"

# 6. Отправить на GitHub

```

## 5. Полезные команды для работы с конспектами

### Просмотр истории
```bash
# Показать все коммиты (кратко)
git log --oneline

# Показать последние 5 коммитов
git log --oneline -5

# Показать историю в виде графа
git log --oneline --graph --all
```

### Просмотр изменений
```bash
# Показать изменения перед добавлением
git diff

# Показать, что будет в коммите
git diff --staged
```

### Отмена изменений
```bash
# Отменить добавление файла
git reset HEAD filename.md

# Отменить последний коммит (сохраняя изменения)
git reset --soft HEAD~1

# Отменить последний коммит (удаляя изменения) — осторожно!
git reset --hard HEAD~1
```

## 8. Настройка Git (первый раз)

```bash

# Проверить настройки
git config --list

# Настроить сохранение пароля/токена
git config --global credential.helper store
```

## 9. Работа с GitHub (push)

### Первоначальная настройка
```bash
# Создать локальный репозиторий
git init

# Добавить удаленный репозиторий
git remote add origin https://github.com/EliantProtocol/linux-notes.git

# Проверить
git remote -v
```

### Отправка на GitHub
```bash
# Первый push
git push -u origin main   # или master

# Последующие push
git push
```

### Если используется токен вместо пароля
```bash
# При запросе пароля вставляете Personal Access Token
Username: ElianProtocol
Password: ghp_ваш_токен_здесь
```

## 11. Частые ошибки и их решение

| Ошибка | Решение |
|--------|---------|
| `git: команда не найдена` | Установить Git: `sudo apt install git` |
| `Author identity unknown` | Настроить `user.name` и `user.email` |
| `src refspec main не соответствует` | Проверить имя ветки: `git branch` |
| `Authentication failed` | Использовать Personal Access Token вместо пароля |
| `failed to push` | Сделать `git pull` сначала, если есть изменения на GitHub |
