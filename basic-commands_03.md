# Конспект: Текстовые редакторы (nano и vim)

## 1. Зачем нужен текстовый редактор в терминале

На серверах нет графического интерфейса, поэтому для редактирования файлов (конфигов, скриптов) нужны **консольные редакторы**.  
Два основных:
- **nano** — простой, интуитивный.
- **vim** — мощный, но со сложным управлением (важно знать, как выйти).

---

## 2. nano — основной редактор

### Интерфейс и нотация
- `^X` означает **Ctrl+X**
- `M-U` означает **Alt+U** (Meta)

Экран разделён:
- **верхняя строка** — имя файла
- **основная область** — содержимое
- **нижние две строки** — подсказки по командам

### Основные команды nano

| Команда | Действие |
|---------|----------|
| `^O` | Сохранить файл |
| `^X` | Выйти из nano |
| `^K` | Вырезать (удалить) строку |
| `^U` | Вставить вырезанную строку |
| `^W` | Найти текст |
| `^\` | Найти и заменить |
| `^_` | Перейти к строке по номеру |
| `Alt+U` | Отменить действие (undo) |
| `Alt+N` | Вкл/выкл номера строк |

> **Стандартный сценарий:** `^X` → `Y` → `Enter` — сохранить и выйти.

### Полезные приёмы

```
# Открыть файл с номерами строк
nano -l notes.txt

# Открыть файл на определённой строке
nano +42 config.txt

# Включить подсветку синтаксиса (добавить в ~/.nanorc)
include "/usr/share/nano/*.nanorc"
```

## 3. vim — минимальный обзор

### Два режима (главное отличие от nano)

```
Нормальный режим (Normal)  ←── vim запускается здесь
       │                        (клавиши = команды)
       │ i — войти в режим вставки
       ▼
Режим вставки (Insert)     ←── здесь можно печатать текст
       │
       │ Esc — вернуться в нормальный режим
       ▼
Нормальный режим
```

> **Правило:** если что-то пошло не так — нажмите **`Esc`**.

### Минимальный набор команд

| Действие | Команда |
|----------|---------|
| Войти в режим вставки | `i` |
| Вернуться в нормальный режим | `Esc` |
| Сохранить | `:w` + `Enter` |
| Выйти | `:q` + `Enter` |
| Сохранить и выйти | `:wq` + `Enter` |
| Выйти без сохранения | `:q!` + `Enter` |

> **Спасательная команда:** если vim открылся и вы не знаете, что делать —  
> `Esc` → `:q!` → `Enter` (выйти без сохранения).

### Дополнительные команды (нормальный режим)

| Команда | Действие |
|---------|----------|
| `x` | Удалить символ под курсором |
| `dd` | Удалить строку |
| `u` | Отменить действие |

### Смена редактора по умолчанию

Если программы (`git`, `crontab`) открывают vim, а вы хотите nano:

```
export EDITOR=nano                    # временно, на сессию
echo 'export EDITOR=nano' >> ~/.bashrc # навсегда
```

## 4. Практические приёмы

### Работа с файлами проекта

```
# Открыть конфиг в nano
nano wordpress-project/config/nginx.conf

# Открыть HTML-файл в vim
vim wordpress-project/site/index.html
```

### Типичный сценарий в nano
1. Открыть файл: `nano file.conf`
2. Внести изменения (печатать текст)
3. Найти нужное место: `^W` (поиск)
4. Сохранить: `^O` → `Enter`
5. Выйти: `^X`

### Типичный сценарий в vim
1. Открыть файл: `vim file.conf`
2. Войти в режим вставки: `i`
3. Внести изменения
4. Вернуться в нормальный режим: `Esc`
5. Сохранить и выйти: `:wq` + `Enter`

## 6. Итог: что нужно запомнить

1. **nano** — основной редактор для повседневной работы:
   - `^O` — сохранить
   - `^X` — выйти
   - `^W` — найти
   - Просто печатайте текст

2. **vim** — нужно знать, как выйти:
   - `Esc` → `:q!` → `Enter` — выход без сохранения
   - `Esc` → `:wq` → `Enter` — сохранить и выйти
   - `i` — начать печатать

3. **Правило жизни:** если vim открылся неожиданно — нажмите `Esc`, затем `:q!` и `Enter`.
