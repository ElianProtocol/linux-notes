## 1. Зачем это нужно

В реальной работе приходится обрабатывать **тысячи строк** логов, находить ошибки, подсчитывать статистику. Linux предоставляет набор небольших утилит, каждая из которых делает **одно простое дело**. Их сила — в возможности **соединять** в цепочки через конвейер (`|`).

---

## 2. Перенаправление ввода-вывода

### Три стандартных потока

| Поток | Номер | Описание | По умолчанию |
|-------|-------|----------|--------------|
| **stdin** | 0 | Ввод данных | Клавиатура |
| **stdout** | 1 | Вывод результата | Экран |
| **stderr** | 2 | Вывод ошибок | Экран |

### Операторы перенаправления

| Оператор | Действие | Пример |
|----------|----------|--------|
| `>` | Перезаписать файл | `ls > list.txt` |
| `>>` | Добавить в конец файла | `echo "text" >> file.txt` |
| `2>` | Перенаправить ошибки | `ls /no 2> errors.txt` |
| `2>&1` | Направить stderr туда же, куда stdout | `cmd > out.txt 2>&1` |
| `2>/dev/null` | Подавить ошибки | `ls /no 2>/dev/null` |

> **`/dev/null`** — «чёрная дыра». Всё, что туда пишется, исчезает.

### Конвейер (pipe) `|`

Соединяет **stdout** левой команды с **stdin** правой:

```bash
# Посчитать количество файлов
ls /etc | wc -l

# Цепочка из трёх команд
ls /etc | sort | head -5
```

## 3. Подготовка: пример лог-файла nginx

Создайте файл с логами для практики:

```bash
nano ~/wordpress-project/logs/access.log
```

**Формат строки лога:**

```
192.168.1.10 - - [15/Mar/2026:10:12:01 +0300] "GET / HTTP/1.1" 200 3421
│             │ │                            │                  │   │
IP-адрес     - - дата и время                запрос             код размер
```

**Коды HTTP:**

| Код | Значение |
|-----|----------|
| `200` | OK |
| `302` | Перенаправление |
| `403` | Доступ запрещён |
| `404` | Не найдено |

## 4. Основные утилиты

### wc — подсчёт строк, слов, байт

```bash
wc access.log          # строки, слова, байты
wc -l access.log       # только строки
ls /etc | wc -l        # количество файлов
```

| Флаг | Что считает |
|------|-------------|
| `-l` | строки |
| `-w` | слова |
| `-c` | байты |

### sort — сортировка

```bash
sort access.log                           # алфавитная
sort -r access.log                        # обратная
sort -n -k 9 access.log                   # числовая по 9-му полю
```

| Флаг | Действие |
|------|----------|
| `-r` | обратный порядок |
| `-n` | числовая сортировка |
| `-k N` | сортировать по N-му полю |
| `-u` | убрать дубликаты (как `sort \| uniq`) |

### uniq — удаление дубликатов

**Важно:** удаляет только **подряд идущие** одинаковые строки → почти всегда используется после `sort`.

```bash
cut -d ' ' -f 1 access.log | sort | uniq          # уникальные IP
cut -d ' ' -f 1 access.log | sort | uniq -c       # с подсчётом
```

| Флаг | Действие |
|------|----------|
| `-c` | показать количество повторений |
| `-d` | только повторяющиеся |
| `-u` | только уникальные |

### cut — извлечение столбцов

```bash
# По разделителю
cut -d ' ' -f 1 access.log          # первое поле (IP)
cut -d ' ' -f 1,9 access.log        # 1-е и 9-е поля
cut -d ' ' -f 1-3 access.log        # поля 1-3

# По символам
cut -c 1-12 access.log              # первые 12 символов
```

### grep — поиск по шаблону

```bash
grep '404' access.log                     # строки с 404
grep -c '403' access.log                  # количество
grep -v '200' access.log                  # всё, кроме 200
grep -n 'wp-login' access.log             # с номерами строк
grep -r 'server' ~/config/                # рекурсивный поиск
```

| Флаг | Действие |
|------|----------|
| `-i` | игнорировать регистр |
| `-c` | только количество |
| `-n` | номера строк |
| `-v` | инвертировать (показать НЕ содержащие) |
| `-r` | рекурсивно |
| `-E` | расширенные регулярные выражения |

## 5. Регулярные выражения (базовый уровень)

### Основные метасимволы

| Символ | Значение | Пример | Совпадёт с |
|--------|----------|--------|------------|
| `.` | любой один символ | `a.c` | `abc`, `a1c` |
| `*` | предыдущий символ 0+ раз | `ab*c` | `ac`, `abc`, `abbc` |
| `^` | начало строки | `^192` | строки, начинающиеся с 192 |
| `$` | конец строки | `php$` | строки, заканчивающиеся на php |
| `[]` | любой символ из набора | `[0-9]` | любая цифра |
| `\` | экранирование | `\.php` | точка и php (буквально) |

### Расширенные (`-E`)

| Символ | Значение | Пример |
|--------|----------|--------|
| `+` | 1+ раз | `ab+c` → `abc`, `abbc` |
| `?` | 0 или 1 раз | `colou?r` → `color`, `colour` |
| `\|` | или | `cat\|dog` |
| `()` | группировка | `(ab)+` |

```bash
# Найти ошибки 403 или 404
grep -E '(403|404)' access.log

# Найти коды 4xx или 5xx
grep -E '" (4[0-9]{2}|5[0-9]{2}) ' access.log
```

## 6. sed — потоковая замена

```bash
sed 's/старое/новое/' file           # замена первого вхождения в строке
sed 's/старое/новое/g' file          # замена всех вхождений
sed -i 's/старое/новое/g' file       # изменить файл (осторожно!)

sed '/403/d' file                    # удалить строки с 403
sed -n '5,10p' file                  # вывести строки 5-10
```

> **По умолчанию** sed выводит результат на экран, не меняя файл.  
> `-i` — изменение файла (отменить нельзя!).

### sed в конвейере

```bash
# Извлечь URL и убрать HTTP/1.1
cut -d '"' -f 2 access.log | sed 's/ HTTP\/1.1//'
```

## 7. awk — обработка по столбцам

`awk` автоматически разбивает строку на поля по пробелам:

| Переменная | Значение |
|------------|----------|
| `$1` | Первое поле |
| `$2` | Второе поле |
| `$NF` | Последнее поле |
| `$0` | Вся строка |

```bash
awk '{print $1, $9}' access.log        # IP и код ответа
awk '{print $1, $NF}' access.log       # IP и размер

# Фильтрация
awk '$9 == 404' access.log             # строки с кодом 404
awk '$9 == 403 {print $1}' access.log  # IP с 403

# Суммирование
awk '{sum += $NF} END {print sum}' access.log
```

### Указание разделителя

```bash
awk -F: '{print $1}' /etc/passwd       # поля разделены :
```

## 8. Комбинирование: реальные задачи

### Найти 3 самых активных IP

```bash
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -3
```

| Шаг | Команда | Действие |
|-----|---------|----------|
| 1 | `awk '{print $1}'` | извлечь IP |
| 2 | `sort` | сгруппировать одинаковые |
| 3 | `uniq -c` | подсчитать повторения |
| 4 | `sort -rn` | отсортировать по убыванию |
| 5 | `head -3` | взять топ-3 |

### Подсчитать количество ошибок (4xx и 5xx)

```bash
grep -E '" (4[0-9]{2}|5[0-9]{2}) ' access.log | wc -l
```

### Сохранить ошибки в отдельный файл

```bash
grep -E '" (4[0-9]{2}|5[0-9]{2}) ' access.log > errors.log
```

### Общий объём переданных данных

```bash
awk '{sum += $NF} END {print sum, "байт"}' access.log
```

## 9. Справочник команд

| Задача | Команда |
|--------|---------|
| Количество строк | `wc -l file` |
| Уникальные IP | `cut -d ' ' -f 1 file \| sort \| uniq` |
| Поиск по шаблону | `grep 'pattern' file` |
| Замена текста | `sed 's/old/new/g' file` |
| Извлечь столбцы | `awk '{print $1, $2}' file` |
| Сортировка с подсчётом | `sort \| uniq -c \| sort -rn` |
| Подавить ошибки | `command 2>/dev/null` |

## 10. Итоги

**Что нужно запомнить:**

| № | Что узнали | Ключевое |
|---|------------|----------|
| 1 | **Три потока** | stdin (0), stdout (1), stderr (2) |
| 2 | **Перенаправление** | `>` (перезапись), `>>` (добавление), `2>` (ошибки) |
| 3 | **Конвейер `|`** | Связывает stdout левой команды с stdin правой |
| 4 | **wc -l** | Подсчёт строк |
| 5 | **sort** | Сортировка |
| 6 | **uniq -c** | Подсчёт повторений |
| 7 | **grep** | Фильтрация |
| 8 | **cut / awk** | Извлечение полей |
| 9 | **sed** | Замена |
| 10 | **Регулярные выражения** | `.` (любой символ), `*` (0+), `^` (начало), `$` (конец) |

**📌 Ключевое правило администратора:** Комбинируйте простые инструменты через `|` для решения сложных задач. Одна утилита — одно действие.

**📌 Типичная ошибка:** Забыли, что `uniq` работает только с отсортированными данными

```bash
# ❌ НЕПРАВИЛЬНО — uniq не удалит все дубликаты
cat file | uniq -c

# ✅ ПРАВИЛЬНО — сначала сортировка
cat file | sort | uniq -c
```

**📌 Полезные комбинации для анализа логов:**

```bash
# Топ-10 IP по количеству запросов
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -10

# Количество успешных запросов (200)
grep -c '" 200 ' access.log

# Все уникальные URL с ошибкой 404
awk '$9 == 404 {print $7}' access.log | sort -u
```
