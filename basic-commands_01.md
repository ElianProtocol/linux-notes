## 1. Что такое Linux

**Linux** — это ядро операционной системы.  
Для повседневной работы используется **дистрибутив** — ядро + набор программ и утилит.

| Компонент | Описание |
|-----------|----------|
| **Ядро (kernel)** | Управляет оборудованием, памятью, процессами |
| **Дистрибутив** | Ядро + пользовательские программы (GNU утилиты, менеджер пакетов и т.д.) |

В курсе используется **Debian 13 (trixie)** — стабильный и широко распространённый дистрибутив.

> **Интересный факт:** Linux работает на серверах, Android, суперкомпьютерах и встроенных системах.

## 2. Подключение по SSH

**SSH (Secure Shell)** — протокол безопасного удалённого доступа к серверу по сети.

### Из Windows (встроенный SSH-клиент)

```bash
ssh student@192.168.1.100
```

| Элемент | Значение |
|---------|----------|
| `student` | Имя пользователя на сервере |
| `192.168.1.100` | IP-адрес сервера |

**При первом подключении:**

```
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

Введите `yes` — это подтверждает ключ хоста.

> **Важно:** При вводе пароля символы не отображаются на экране — это нормально, так работает защита.

### Завершение сеанса

```bash
exit
```

Или нажмите `Ctrl + D`

## 3. Структура командной строки

```
student@debian:~$
```

| Элемент | Значение |
|---------|----------|
| `student` | Текущий пользователь |
| `debian` | Имя хоста (компьютера) |
| `~` | Текущий каталог (`~` = домашний каталог пользователя) |
| `$` | Признак обычного пользователя (`#` — root, суперпользователь) |

---

## 4. Основные команды

| Команда | Назначение | Пример |
|---------|------------|--------|
| `whoami` | Имя текущего пользователя | `whoami` → `student` |
| `hostname` | Имя компьютера | `hostname` → `debian` |
| `pwd` | Текущий каталог (print working directory) | `pwd` → `/home/student` |
| `ls` | Содержимое каталога (list) | `ls /` — показать корневые каталоги |
| `cd` | Смена каталога (change directory) | `cd /tmp`, `cd` (возврат домой) |
| `clear` | Очистка экрана | `clear` |
| `exit` | Выход из сеанса | `exit` |

| Команда | Назначение |
|---------|------------|
| `whoami` | Показать имя текущего пользователя |

| `ls` | Показать содержимое каталога |

| `cd` | Перейти в домашний каталог |
| `clear` | Очистить экран |
| `exit` | Выйти из сеанса SSH |

## 6. Итоги

**Что нужно запомнить:**

| № | Что узнали | Ключевое |
|---|------------|----------|
| 1 | **Linux** | Это ядро, а не вся ОС |
| 2 | **Дистрибутив** | Ядро + набор программ (например, Debian) |
| 3 | **SSH** | Протокол для безопасного удалённого доступа |
| 4 | **Структура строки** | `пользователь@хост:каталог$` |
| 5 | **~ (тильда)** | Сокращение для домашнего каталога |
| 6 | **$ vs #** | `$` — обычный пользователь, `#` — root |
| 7 | **Первые команды** | `whoami`, `hostname`, `pwd`, `ls`, `cd`, `clear`, `exit` |

**📌 Ключевое правило администратора:** При подключении по SSH к новому серверу всегда проверяйте, под каким пользователем вы зашли (`whoami`), где находитесь (`pwd`) и какой у вас уровень прав (`$` или `#`).

**📌 Типичная ошибка новичка:** Попытка ввести команды, когда курсор мигает после ввода пароля

```bash
# ❌ Пароль не отображается — это нормально
student@192.168.1.100's password: █

# ✅ Просто введите пароль и нажмите Enter, даже если ничего не видно
```
