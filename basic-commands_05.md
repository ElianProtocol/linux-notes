## 1. Зачем нужны скрипты

**Скрипт** — текстовый файл с последовательностью команд, которые выполняются автоматически. Позволяет автоматизировать повторяющиеся задачи вместо ручного ввода команд каждый раз.

---

## 2. Переменные окружения и PATH

### Что такое переменные окружения

Хранят настройки системы и пользователя. Доступны всем запускаемым программам.

| Переменная | Описание |
|------------|----------|
| `HOME` | Домашний каталог |
| `USER` | Имя пользователя |
| `PATH` | Где искать программы |
| `EDITOR` | Редактор по умолчанию |

```bash
env                    # все переменные
echo $HOME             # вывести значение
export GREETING="Hi"   # создать и экспортировать
```

> **Важно:** `=` без пробелов! `NAME="value"` — правильно, `NAME = "value"` — ошибка.

### PATH — поиск команд

```bash
echo $PATH             # список каталогов через :
which ls               # где находится программа
```

Если программа не в PATH → запускать с путём: `./script.sh`

### Файл ~/.bashrc

Выполняется при каждом входе в систему. Добавляйте настройки сюда:

```bash
echo 'export EDITOR=nano' >> ~/.bashrc
source ~/.bashrc       # применить изменения
```

## 3. Первый скрипт

```bash
#!/bin/bash           # shebang — указывает интерпретатор

echo "Привет!"
date
```

```bash
chmod u+x script.sh   # добавить право на выполнение
./script.sh           # запуск
# или
bash script.sh        # запуск без chmod
```

> **Shebang** (`#!`) — всегда первая строка. Указывает, какой интерпретатор использовать.

---

## 4. Переменные в скриптах

```bash
NAME="мир"
echo "Привет, $NAME"        # двойные кавычки — подстановка
echo 'Привет, $NAME'        # одинарные — без подстановки

# Фигурные скобки — границы переменной
FILE="access"
echo "${FILE}_log"           # access_log
```

### Подстановка команд

```bash
TODAY=$(date +%Y-%m-%d)      # результат команды в переменную
FILES=$(ls | wc -l)
```

## 5. Аргументы скрипта

```bash
./script.sh arg1 arg2 arg3
```

| Переменная | Значение |
|------------|----------|
| `$0` | Имя скрипта |
| `$1`, `$2`... | Аргументы |
| `$#` | Количество аргументов |
| `$@` | Все аргументы |

```bash
#!/bin/bash
echo "Скрипт: $0"
echo "Привет, $1!"
echo "Всего аргументов: $#"
```

## 6. Условия (if)

### Синтаксис if

```bash
if [ условие ]; then
    команды
elif [ другое_условие ]; then
    команды
else
    команды
fi
```

> **Пробелы обязательны!** `[ -f file ]` — правильно.

### Основные проверки

| Условие | Что проверяет |
|---------|---------------|
| `[ -f file ]` | Файл существует |
| `[ -d dir ]` | Каталог существует |
| `[ -e path ]` | Путь существует |
| `[ -z "$str" ]` | Строка пуста |
| `[ "$a" = "$b" ]` | Строки равны |
| `[ "$a" -eq "$b" ]` | Числа равны |
| `[ "$a" -gt "$b" ]` | a > b |

### Проверка результата команды

```bash
if grep -q "pattern" file; then
    echo "Найдено"
fi
```

## 7. Циклы

### for — перебор списка

```bash
for ITEM in список; do
    команды
done
```

**Примеры:**

```bash
# Перебор явного списка
for CITY in Москва Петербург; do
    echo "Город: $CITY"
done

# Перебор файлов
for FILE in ~/config/*; do
    echo "Файл: $FILE"
done

# Перебор аргументов
for ARG in "$@"; do
    echo "Аргумент: $ARG"
done
```

### while — цикл с условием

```bash
while [ условие ]; do
    команды
done
```

**Примеры:**

```bash
# Чтение файла построчно
while read LINE; do
    echo "Строка: $LINE"
done < file.txt

# Счётчик
COUNT=5
while [ $COUNT -gt 0 ]; do
    echo $COUNT
    COUNT=$((COUNT - 1))
done
```

`$((...))` — арифметические вычисления.

## 8. Коды возврата

- **0** — успех
- **не 0** (1, 2, 127...) — ошибка

```bash
ls /etc
echo $?          # код возврата последней команды
```

### exit в скриптах

```bash
#!/bin/bash
if [ $# -eq 0 ]; then
    echo "Ошибка: нужен аргумент" >&2
    exit 1
fi
exit 0
```

> `>&2` — перенаправление в stderr (поток ошибок).

## 9. Практика: скрипт резервного копирования

```bash
#!/bin/bash

PROJECT_DIR="$HOME/wordpress-project"
BACKUP_DIR="${1:-$HOME/backups}"          # аргумент или значение по умолчанию
DATE=$(date +%Y-%m-%d_%H-%M-%S)
ARCHIVE_NAME="wordpress-backup-${DATE}.tar.gz"

# Проверка существования каталога проекта
if [ ! -d "$PROJECT_DIR" ]; then
    echo "Ошибка: каталог $PROJECT_DIR не найден" >&2
    exit 1
fi

# Создание каталога для бэкапов
if [ ! -d "$BACKUP_DIR" ]; then
    mkdir -p "$BACKUP_DIR"
fi

# Создание архива
tar czf "$BACKUP_DIR/$ARCHIVE_NAME" -C "$HOME" wordpress-project

# Проверка результата
if [ $? -eq 0 ]; then
    echo "Готово! Размер: $(du -h "$BACKUP_DIR/$ARCHIVE_NAME" | cut -f1)"
else
    echo "Ошибка при создании бэкапа!" >&2
    exit 1
fi
```

### Ключевые конструкции

| Конструкция | Описание |
|-------------|----------|
| `${1:-значение}` | Значение по умолчанию, если аргумент не передан |
| `tar czf` | Создание сжатого архива |
| `$?` | Проверка кода возврата |
| `du -h` | Размер файла в читаемом формате |

## 10. Справочник команд

| Конструкция | Назначение |
|-------------|------------|
| `#!/bin/bash` | Shebang |
| `chmod u+x file` | Сделать исполняемым |
| `$VAR` | Значение переменной |
| `${VAR:-default}` | Значение по умолчанию |
| `$(command)` | Подстановка команды |
| `$1`, `$2`... | Аргументы скрипта |
| `$#` | Количество аргументов |
| `$@` | Все аргументы |
| `$?` | Код возврата |
| `if [ условие ]; then ... fi` | Условие |
| `for i in list; do ... done` | Цикл for |
| `while [ условие ]; do ... done` | Цикл while |
| `exit N` | Завершить с кодом N |
| `>&2` | Перенаправить в stderr |

## 11. Итоги

**Что нужно запомнить:**

| № | Что узнали | Ключевое |
|---|------------|----------|
| 1 | **Shebang** | `#!/bin/bash` — обязателен в начале скрипта |
| 2 | **Переменные** | `VAR="value"` (без пробелов у `=`), вызов через `$VAR` |
| 3 | **Кавычки** | Двойные (`"`) — подстановка, одинарные (`'`) — без подстановки |
| 4 | **PATH** | Определяет, где искать программы → для своих скриптов нужен `./` |
| 5 | **Аргументы** | `$1`, `$2`, `$#` — количество, `$@` — все |
| 6 | **Условия** | Пробелы в `[ ]` обязательны, `-f` для файлов, `-d` для каталогов |
| 7 | **Циклы** | `for` для перебора списка, `while` для повторения по условию |
| 8 | **Коды возврата** | `0` = успех, `exit 1` для ошибок |
| 9 | **Лучшая практика** | Сообщения об ошибках отправлять в stderr: `>&2` |

**📌 Ключевое правило администратора:** Всегда проверяйте код возврата после критических операций (создание бэкапов, копирование файлов, изменение конфигурации).

**📌 Типичная ошибка:** Пробелы в условии if

```bash
# ❌ БУДЕТ ОШИБКА
if [-f file]; then

# ✅ ПРАВИЛЬНО
if [ -f file ]; then
```

**📌 Отладка скриптов:**

```bash
# Режим отладки — показывает каждую команду перед выполнением
bash -x script.sh

# Или внутри скрипта
#!/bin/bash
set -x  # включить отладку
команды
set +x  # выключить отладку
```
