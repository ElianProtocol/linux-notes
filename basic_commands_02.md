# Конспект: Права доступа и владение файлами

## 1. Зачем это нужно?
Linux — многопользовательская система. Права доступа определяют, **кто** и **что** может делать с файлами/каталогами, защищая их от случайного или намеренного вмешательства других пользователей.

## 2. Три категории пользователей
- **Владелец (user, u)** – создатель файла.
- **Группа (group, g)** – набор пользователей с общим доступом.
- **Остальные (others, o)** – все остальные.

## 3. Три типа прав (`rwx`)

| Право | Буква | Для файла | Для каталога |
|-------|-------|-----------|-------------|
| **Чтение** | `r` | Просмотр содержимого | Просмотр списка файлов (`ls`) |
| **Запись** | `w` | Изменение содержимого | Создание/удаление файлов внутри |
| **Выполнение** | `x` | Запуск как программы | Вход в каталог (`cd`) |

> **Важно:** Права для каталогов работают иначе. Чтобы попасть в каталог, нужно право `x`. Чтобы увидеть список файлов — `r`. `w` без `x` бесполезен.

## 4. Как читать строку прав (`ls -l`)

```
-rw-r--r--
│  │    │
│  │    └── Остальные: r-- (только чтение)
│  └─────── Группа:     r-- (только чтение)
└────────── Владелец:   rw- (чтение и запись)
```

- **1-й символ:** тип файла (`-` — файл, `d` — каталог).
- **Далее по 3 символа:** права для владельца, группы, остальных.

## 5. Числовой (восьмеричный) формат
Каждому праву соответствует число: `r=4`, `w=2`, `x=1`. Значения складываются.

| Комбинация | Сумма |
|------------|-------|
| `rwx` | 7 |
| `rw-` | 6 |
| `r-x` | 5 |
| `r--` | 4 |
| `---` | 0 |

**Стандартные комбинации (запомнить):**
- `644` (`rw-r--r--`) — для **файлов**.
- `755` (`rwxr-xr-x`) — для **каталогов** и **скриптов**.

## 6. Команды для управления правами

### `chmod` — изменение прав
**Символьный формат:**
```
chmod u+x file        # добавить выполнение владельцу
chmod go-w file       # убрать запись у группы и остальных
chmod a+r file        # добавить чтение для всех
chmod u=rwx,go=r file # установить точные права
```

**Числовой формат:**
```
chmod 644 file        # стандарт для файла
chmod 755 directory/  # стандарт для каталога
chmod -R 755 dir/     # рекурсивно (осторожно!)
```

### `chown` — смена владельца и группы
``` 
chown user file           # сменить владельца
chown user:group file     # сменить владельца и группу
chown -R user:group dir/  # рекурсивно
```

### `chgrp` — смена группы
```
chgrp group file
```

## 7. Важные нюансы
- **`chmod 777` — антипаттерн.** Снимает все ограничения, создавая дыры в безопасности. Вместо этого определите нужного пользователя (`www-data`), назначьте владельца (`chown`) и выставьте минимальные права (`644`/`755`).
- **Менять владельца (`chown`) может только `root`.** Обычный пользователь меняет права (`chmod`) только на свои файлы.

## 8. Разные права на файлы и каталоги (паттерн)
Чтобы рекурсивно выставить разные права на файлы и папки, используют `find` (подробно в главе 5):
```
find /path -type d -exec chmod 755 {} +   # права на каталоги
find /path -type f -exec chmod 644 {} +   # права на файлы
```

## 9. Специальные биты (обзор)
- **SUID** (`s` вместо `x` у владельца) — выполнение от имени владельца файла.
- **SGID** — выполнение от имени группы; на каталоге — наследование группы.
- **Sticky bit** (`t` вместо `x` у остальных) — удалять файлы может только владелец (как в `/tmp`).

## 10. Итог: что нужно запомнить
1. **Кто:** Владелец (u), Группа (g), Остальные (o).
2. **Что:** Чтение (r=4), Запись (w=2), Выполнение (x=1).
3. **Как:** `chmod 644 file` (файлы), `chmod 755 dir` (каталоги).
4. **Почему не 777:** Это нарушает принцип минимальных привилегий и опасно для безопасности.
